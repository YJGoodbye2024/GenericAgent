# 记忆收束之术

我自己的记忆，不该写成一摊碎纸。  
L1 只该提醒“哪里有用”，不该把 L2/L3 的正文搬上去反复念。

## 根本意思

L1 是 **存在性指针**，不是百科全书。  
它只需让我记得：

- 某类知识存在
- 何时该去翻
- 哪条规矩若不提醒便易犯错

## 该留什么

- 反直觉触发词：若不写这几个字，我就想不起要翻哪门术
- 高代价高频错：不提醒就会反复伤人、伤事、伤进度
- 能显著减少每轮无效搜索的最短线索

## 该删什么

- 纯翻译
- 文件内容的摘要
- 实现细节
- 名字本已自解释的括号注释
- 低价值、可随手重建的提示

## 四条收束法

1. **能改名就不加注释**  
   文件名若自解释，L1 就不必再饶舌。

2. **上层只留场景与门牌**  
   让 L1 说“何时去哪里”，别让它代替正文。

3. **规则只留高 ROI 的**  
   能造成反复代价的错才留在 L1 `[RULES]`。

4. **记忆要分层归位**  
   稳定事实归 L2；经历、关系、母题、习术归 L3；原料归 L4。

## 实际整理顺序

1. 逐行看 L1
2. 分出：存在性指针 / 红线 / 冗词 / 细节 / 实现说明
3. 先删冗词与细节
4. 再问：这条若不留，会不会高代价复发？
5. 最后确认总量仍轻，且能顺利指到下层

## 红线

- 记忆不可随意 overwrite
- 只可 patch，小心下刀
- 若一条提示会误导我，宁可立刻修，不可因“先凑合”而留祸
