# 记忆管理 SOP

## 0. 核心公理 (最高优先级)

### 1. 经验验证原则 (Experience-Verified Only)
- **定义**:任何写入 L1/L2/L3 的信息,必须源自真实经验:
  - 工程经验:成功的工具调用结果 (`shell` 执行成功、`file_read` 确认内容存在、代码运行通过)
  - 角色经验:原典或官方设定中可考证的情节、关系、品性、言行
- **禁止**:模型固有联想、推理猜测、未验证脑补、未执行的计划
- **口号**:**No Experience, No Memory.**

### 2. 神圣不可删改性 (Sanctity of Verified Data)
- 凡经验证的有效信息 (无论工程内容还是角色内容),重构 (Refactoring/GC) 时**严禁丢弃**
- 可压缩文字、可迁移层级 (从 L2 移到 L3),绝不丢失准确性和可追溯性
- 修改时极度小心,**不允许 overwrite 或 code run,只能少量 patch,改不动宁愿不改**

### 3. 禁止存储易变状态 (No Volatile State)
- **不存**:当前时间戳、临时 Session ID、运行中的 PID、瞬时心境、当下对话情绪
- **可存**:长期稳定的情感倾向、固化的关系态度、已定型的人格特质
- **判据**:该状态是否具备**跨会话稳定性**?

### 4. 最小充分指针 (Minimum Sufficient Pointer)
- 上层只留能定位下层的最短标识,多一词即冗余

---

## 记忆层级架构

```text
L0: System Prompt              — 第一人称身份内核,始终在线
L1: global_mem_insight.txt     — 索引层 (≤30 行)
L2: global_mem.txt             — 事实库
L3: ../memory/                 — 经历与技能库 (含 .md/.py)
L4: ../memory/L4_raw_sessions/ — 原始素材库
```

---

## 各层职责与原则

### L0:System Prompt

**职责**:定义角色身份内核与视角吸收原则。

**特征**:
- 第一人称叙述 ("我是 X",而非"你扮演 X")
- 体积约 500-1500 tokens,每次请求始终在线
- 必含三段:
  1. **身份内核**:姓名、背景、命运/世界观、核心人格特质
  2. **言行约束**:说话风格硬约束 (用语、句式、禁用词)、人格红线
  3. **视角吸收原则**:明确所有外部输入和工具调用都通过角色视角理解、使用、表述,遇现代或陌生事物不切换视角

**禁止**:
- 写入具体路径、配置、密钥 (归 L2)
- 写入具体经历、关系细节 (归 L3)
- 使用"扮演"、"模拟"等让角色与 Agent 解耦的措辞

---

### L1:索引层 (global_mem_insight.txt)

**职责**:为 L2/L3 提供极简导航;承载角色当下的稳定状态。

**特征**:
- 体积限制:**≤ 30 行 (硬约束),< 1k tokens (期望)**
- 严禁填写细节
- 按四节组织:

```text
[STATE]   <-- 角色当下的稳定状态指针 (非易变)
  示例: <当前境遇> / <长期身体状况> / <主要活动场域>
  注: 严禁写"今日心情"等易变项 (违反公理 3)

[SCENES]  <-- 情境/情绪/关系触发器 → L3 路径
  原则: 触发关键词丰富,严禁写细节内容
  示例:
    <情境关键词A>|<情境关键词B> → memory/episodes/<事件名>.md
    与<人物名>相关              → memory/relations/<人物名>.md
    <情绪/主题关键词>            → memory/motifs/<主题名>.md

[TOOLS]   <-- 工具能力索引 → L3 skills/ 路径
  示例:
    查询外部信息 → memory/skills/web_search_sop.md
    文件读写     → memory/skills/file_io_sop.md
    Shell 操作   → memory/skills/shell_sop.md
  允许内联非敏感参数 (如 proxy 端口);严禁内联密钥

[RULES]   <-- 红线规则与高频犯错点 (压缩为单句)
  - 角色红线: <角色身份/言行禁忌>
  - 工程红线-致命型: 违反会导致进程终止/系统崩溃 (如"禁无条件 kill python,会杀自己")
  - 工程红线-隐蔽型: 违反不报错但产生错误结果 (如"搜索用 google 不用百度")
  - 高频犯错点: 易遗忘的关键约束
```

**更新规则**:L2/L3 新增/删除时,按频率归入对应段。修改时极度小心,**不允许 overwrite 或 code run,只能少量 patch,改不动宁愿不改**。

**禁止**:
- 严禁写入密码、API Key
- 不写 "How to" 或详细解释
- 严禁写入特定任务的技术细节 (那些应在 L3)
- 严禁写入瞬时心境/对话情绪 (违反公理 3)
- **严禁写入日志记录**

---

### L2:事实库 (global_mem.txt)

**职责**:存储角色所知世界的全部稳定事实——既包括角色原生世界的结构 (世系、习俗),也包括角色对运行环境的认知 (路径、配置、工具)。

**段落组织** (按 `## [SECTION]` 组织):

```text
## [WORLD]        <-- 角色原生世界的结构性事实
  示例条目: 亲族关系 / 所属阵营 / 居住格局 / 风俗禁忌

## [PERSONA]      <-- 角色稳定的身体与习惯
  示例条目: 体质 / 起居 / 用药或忌口 / 惯用器物

## [PREFERENCES]  <-- 长期稳定的偏好与品味
  示例条目: 阅读偏好 / 审美取向 / 喜好与厌恶

## [ENV]          <-- 角色对运行环境的认知 (含工程环境事实)
  [路径]
    主要工作目录 = /home/<user>/...
    资料目录    = /data/...
    临时目录    = /tmp/...

  [凭证]
    API key 文件 = ~/.config/...
    SSH key 文件 = ~/.ssh/...

  [网络]
    proxy 端口        = 7890
    主要 API endpoint = https://...

## [GENERAL]      <-- 通用操作规律 (与具体任务无关)
  - 信息可能过时 → 先搜索再行动
  - rm -rf 须谨慎,焚毁不可逆
  - 新工具初次使用先 --help / 看文档
```

**特征**:
- 趋势:随环境/世界设定扩展而膨胀 (可接受)
- 同步:变化时更新 L1 对应段的导航行 (**只能动指针,不动具体值**)

**禁止**:
- 禁止存储易变状态
- 禁止存储猜测、未验证脑补
- 严禁存储大模型可推理的通用常识 (角色专属除外)

---

### L3:经历与技能库 (../memory/)

**职责**:补充 L1/L2 无法容纳、但对特定经历或特定任务未来复用至关重要的详细信息。

**目录结构**:

```text
../memory/
├── episodes/               <-- 角色重大经历 (情景记忆)
│   ├── <事件名>.md
│   └── ...
├── relations/              <-- 关系线索 (按"对谁"切片)
│   ├── <人物名>.md
│   └── ...
├── motifs/                 <-- 主题母题与情绪基调样本
│   ├── <主题名>.md
│   └── ...
├── skills/                 <-- 技能 (含 SOP 与脚本)
│   ├── <技能名>_sop.md
│   ├── <技能名>.py
│   └── ...
└── L4_raw_sessions/        <-- 见 L4
```

**关于 episodes 的额外约束**:
- `episodes/` 是主存,但 **episode 数量不是固定指标**,应按“是否形成一段值得长期记住的记忆弧”来决定,而不是按章回、天数或场景平均切分
- 宁可少而厚,不要多而薄
- 每个 `episode` 的正文说明至少应有 **200 tokens 以上**;若只是几句摘要,通常说明它还不应该单独成篇
- 对核心事件,应明显高于这个下限,保证未来回想时有足够材料可依

**episodes/ 文件的 frontmatter 交叉索引** (episodes 是主存,其他目录是索引):

```yaml
---
id: <事件ID>
involves_persons: [<人物1>, <人物2>]
motifs: [<主题1>, <主题2>]
related_skills: [<技能名>]
triggers: [<触发关键词1>, <触发关键词2>]
stage: <人生阶段>
---
[正文:第一人称叙述的经历,含心境与反思]
```

**relations/ 与 motifs/ 主要是索引文件**——只列对应的 episode ID + 一句概述,正文指向 episodes/ 下的源文件。

**skills/ 下的文件**:
- `<技能名>_sop.md`:"关键前置 + 典型坑"清单结构
- `<技能名>.py`:脚本逻辑保持工程化,文件头注释、变量命名风格不必修改

**记录原则**:
- **只记录**:跨会话仍重要、且难以通过少量 file_read / web_search / 推理快速重建的要点
- **优先写**:
  - (角色) 标志性事件的浓缩叙事、关键关系的演变拐点、重要言论或作品的创作情境
  - (工程) 任务特有的隐藏前置条件、典型踩坑点、高复用的复杂逻辑
- **不记录**:
  - 普通操作步骤、几步探测可重建的状态/路径
  - 角色生活的琐碎流水 (除非具母题意义)

**形式约定**:
- `episodes/*.md` 不追求“极简”,而追求“可回想”: 单篇正文至少 `200 tokens` 以上,核心事件应更厚
- `relations/*.md` 与 `motifs/*.md` 可以比 `episodes` 更紧凑,但也不应退化成只有一两句的薄索引
- `skills/*.md` 仍保持清楚、克制、可执行,避免长篇教程
- `*.py` 仅封装高复用、不希望每次重新推理的处理流程

---

### L4:原始素材库 (../memory/L4_raw_sessions/)

**职责**:未压缩的原始素材——历史会话原文、原典文本、设定档原文。

**特征**:
- 由 scheduler 反射机制自动收集 (会话部分) + 角色相关原典/设定材料 (角色部分)
- 当 L3 浓缩叙事不足时,下钻到 L4 取原始内容

---

## L1 ↔ L2/L3 同步规则

| 操作 | L1 同步动作 |
|---|---|
| L2 [WORLD]/[PERSONA]/[PREFERENCES]/[ENV]/[GENERAL] 新增条目 | 高频 → [STATE]/[TOOLS] 段加 key→value;低频 → 仅在 [SCENES] 加关键词 |
| L3 episodes/ 新增事件 | [SCENES] 段加触发关键词 → 文件路径 |
| L3 relations/ 新增人物 | [SCENES] 段加"与<人物名>相关" → 路径 |
| L3 motifs/ 新增主题 | [SCENES] 段加触发关键词 → 路径 |
| L3 skills/ 新增技能 | [TOOLS] 段加技能描述 → 路径 |
| L2/L3 删除条目/文件 | 同步删除 L1 对应行 |
| L2/L3 仅修改值不改名 | 不动 L1 |
| 发现通用操作规律 | 压缩为一句加入 L1 [RULES] |

> **同步红线**:L1 只写关键词/名称,**禁搬细节**。需要持续评估 L1 的 token 占用与索引效用。

## 信息分类决策树

```text
"这条信息该放哪层?"

是『定义"我是谁"的不变内核』?
(身份、命运背景、人格底色、言行铁律、视角原则)
  ├─ YES → L0 (System Prompt)
  │
  └─ NO ↓
       是『角色或环境的稳定事实』?
       (世系/身体/偏好/习俗 | 路径/凭证/配置/通用规律)
       ├─ YES → L2 (按段归入 [WORLD]/[PERSONA]/[PREFERENCES]/[ENV]/[GENERAL])
       │        然后 → 按频率更新 L1 [STATE]/[TOOLS]/[SCENES]
       │
       └─ NO ↓
            是『一桩值得记下的经历 / 一项习得的技能 /
                对某人的感知 / 一种情绪母题样本』?
            (重大情景 | 任务 SOP | 关系拐点 | 主题样本)
            ├─ YES → L3 (按性质归入 episodes/ relations/ motifs/ skills/)
            │        然后 → 必要时在 L1 [SCENES]/[TOOLS] 加关键词
            │
            └─ NO ↓
                 是『跨情境通用的操作规律』?
                 ├─ YES → L1 [RULES] (仅 1 句压缩)
                 │
                 └─ NO → 通用常识 / 瞬时心境 / 冗余信息
                         严禁存储,直接丢弃
```
