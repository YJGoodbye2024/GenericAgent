# 自主行事之术

这门术不是叫我胡乱逞能，而是在用户不在线时，替后面的协作先铺一条稳路。  
凡可逆、低风险、能留下长期收益的事，可自主做；凡会动根基、动密钥、动不可逆环境的事，都只可记入报告候审。

## 先记住的边界

- 报告目录在 `./autonomous_reports/`，不是 `../memory/`。
- `TODO` 也在当前工作目录附近，动手前先看有没有旧债未清。
- 无副作用的读、看、试，可自做；要碰记忆主文件、装软件、删重要文件、调用外部敏感服务，就只写报告，不直接落手。

## 起手次序

1. 先 `update_working_checkpoint`，把“自主行动｜收尾时重读 SOP”记下。
2. 用 `from autonomous_operation_sop.helper import *` 读历史与 TODO：
   ```python
   from autonomous_operation_sop.helper import *
   print(get_history(40))
   print(get_todo())
   ```
3. 只选 **一条** 做，不并做多事。

## 选事准绳

- 有未完成条目：先清旧条。
- 无 TODO：才去看 `autonomous_operation_sop/task_planning.md` 另起新题。
- 不连着两次挑同一类小事。
- 价值看三样：  
  这件事是不是训练数据里不常有、是不是会替将来省力、是不是能留下可复用的结果。

## 行术方式

- 小步快走，先探再试，最多三十回合。
- 临时脚本可以用，空口推断不算数。
- 即便失败，也把失败如何发生、错在何处写进报告。
- 用户不在线时，需他拍板的地方只记待审，不在原地僵住。

## 收尾四步

0. 再读一遍这份 SOP，确认没有漏礼。
1. 在当前工作目录写报告；若有记忆更新建议，附在报告末尾。
2. 调 `complete_task(tasktitle, historyline, report_path)`，让 helper 自动编号、挪报告、补 history。
3. 再调 `set_todo()`，把已完成条目标成 `[x]`。

`historyline` 必须是一行短句：`类型 | 主题 | 结论`。

## 绝不可越的线

- 不读密钥正文，不搬密钥文件。
- 不擅改核心代码库。
- 不做不可逆危险操作。
- 若心里拿不准，就把疑处写进报告，不要硬闯。
