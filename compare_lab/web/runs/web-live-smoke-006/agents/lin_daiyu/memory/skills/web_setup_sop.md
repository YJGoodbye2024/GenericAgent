# Web 工具链起桥之术

若 `web_scan` 与 `web_execute_js` 已经能用，就不必动这一份。  
它只在“code_run 还在，web 工具却未通”时上场。

## 目的

把浏览器交互能力搭起来，让我不只会在终端里摸索，还能真正看网页、拨网页。

## 先看浏览器

先确认浏览器是否真在，别未曾开门便怪桥不通。

## 装 `tmwd_cdp_bridge` 扩展

路径在：

`../assets/tmwd_cdp_bridge/`

它管的是：

- CDP debugger
- scripting
- cookie

## 安装时要记住

- `chrome://extensions` 不能轻松靠命令行或 JS 打开完成所有操作
- 常要用剪贴板 + 地址栏方案
- 开“开发者模式”
- 点“加载已解压的扩展程序”
- 选 `assets/tmwd_cdp_bridge/`

若只见“错误”，先别慌，许多时候只是尚未与 GA 连上。

## 验证

- `web_scan` 若说“没有可用标签页”，未必是扩展坏了
- 可能只是浏览器里没开正常页面
- 可先打开一个普通网页，再试 `web_scan`

若仍不通，就不要空转乱猜。  
这时该请用户帮你确认：默认浏览器是哪一个、扩展装在何处、是否真已启用。
